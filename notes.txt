## Animation library
- GSAP
    - most used
    - simple 
    - optimazation to an extent by itself

## Basic syntax
    
    gsap.method("element", {properties});

## methods
- gsap.to() -> Goes from starting point(default) to ending.
    - example syntax -> gsap.to(".box", {
                        delay: 0.6,
                        x: 300,
                        duration: 1,
                      });

- gsap.from() -> Goes from ending point to starting point(default).
    - example syntax -> gsap.from(".box", {
                        delay: 0.6,
                        x: 300,
                        duration: 1,
                      });

- gsap.fromTo() -> can give both states starting and ending so the animation will move from given starting point to given ending point.
    - example syntax -> gsap.fromTo(".box", {
                            x: 0,
                        }, {
                            duration: 1,
                            delay: 0.6,
                            x:400
                        });

- gsap.set() -> method is used to immediately set properties of an element without any animation.
    - usually used for setting the initial desired state for any animation to start the animation
    - example syntax -> gsap.set(".box", {
                            x: 200,
                            y: 100,
                            opacity: 0.5,
                            scale: 2
                        });

- to select multiple elements
    - gsap.to([".box", ".box2"], {properties});

- Custome property 
    const obj = {
            a: 0,
        }
    gsap.to(obj, {
        a: 100
    })


- repeat property 
    - repeat: 2 -> it will run the animation 3 time one time it will run default and it will repeat 2 times
    - repeat: -1 -> infite repeat


- stagger -> if a class given and multiple elements have same class it will run the animations one by one in sequence, reverse order random or center
    - It takes time as its value or object if you want to give direction as well
    - example syntax -> gsap.from("h1 span", {
                            yPercent: 100,
                            duration: 1.5,
                            ease: "expo.out",
                            stagger: {
                                each: 0.08,
                                from: "random"
                            },
                            opacity: 0
                        })


- timeline -> we can run sequence of animations with this by creating a timeline and then chaining animations one after another
    - very useful when you want to run different animations in a desired seqence

    - position parameter
        - "<" -> It animate the current element with the previous element chained in the timeline, both will animate simultaneously.
            - "<0.2" -> it means the delay of 0.2s will be added so it will run after the delay of 0.2s of its previous animation in timeline
        - "-=0.3" -> It will run the current animation 0.3s before its previous animation ending time, so it mean ED of prev. animation - 0.3s
        - "+=0.3" -> It will simply add 0.3s in ending time, it will add delay of 0.3s in current animation applied to.
        - labels -> It basically is like giving a lable to animation in position parameter and the animations with same parameters will run at same time.
            for exmaple if i have given A1 and A4 lable "aman" both will run at same time
            - "aman-=0.2" -> this will run the animation of that perticular element run 0.2seconds earlier, it will reduce 0.2 seconds from the starting time of the label starting time
            - "aman+0.2" -> and it will do reverse of above act as delay
